🚀 Data Warehouse Project
📌 Project Overview

This project focuses on building a modern Data Warehouse using SQL, implementing an end-to-end ETL pipeline along with the Medallion Architecture (Bronze, Silver, Gold layers) to transform raw data into meaningful business insights.

🏗️ Data Engineering (Data Warehouse)
🎯 Objective

Develop a scalable data warehouse to consolidate sales data from multiple sources, enabling analytical reporting and better decision-making.

⚙️ Specifications
Data Sources:
Imported data from two systems (ERP & CRM) provided as CSV files.
Data Quality:
Performed data cleaning, handling missing values, and resolving inconsistencies before analysis.
Integration:
Combined multiple datasets into a single, user-friendly data model optimized for analytical queries.
Scope:
Focused on the latest dataset (no historization applied).
Documentation:
Created clear documentation of the data model for both business stakeholders and analytics teams.
🔄 ETL Pipeline
Extract: Data collected from ERP & CRM sources
Transform: Data cleaned, validated, and structured
Load: Data stored into warehouse for analysis
🥉🥈🥇 Medallion Architecture

This project follows a layered architecture to improve data quality step by step:

Bronze Layer (Raw Data):
Stores data in its original format directly from source systems.
Silver Layer (Cleaned Data):
Data is cleaned, validated, and standardized for consistency.
Gold Layer (Business Ready):
Aggregated and transformed data optimized for reporting and analytics.

👉 This approach ensures better data quality, scalability, and performance by refining data across layers.

📊 BI: Analytics & Reporting
🎯 Objective

Develop SQL-based analytics to generate insights into:

👤 Customer Behavior
📦 Product Performance
📈 Sales Trends

These insights help stakeholders make data-driven decisions.
